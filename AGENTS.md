# AGENTS.md — 日本林道データベース

## 1. ミッション

あなたはこのプロジェクトの実装・調査担当エージェントである。

最優先目標は、**2026年7月11〜12日の長野県での実走に使えるMVPを完成させること**。

将来の理想設計より、今週末に実際に使えることを優先する。

## 2. 優先順位

1. 茅野・諏訪周辺の候補林道を地図表示できる状態にする
2. スマホで使いやすくする
3. URLで仲間と共有できるようにする
4. 各林道からGoogle Mapsナビを開始できるようにする
5. 実走候補3〜5本のカルテを作る
6. 実走後にデータを更新しやすくする

## 3. 今週末MVPの既定技術方針

特別な理由がない限り、次で実装する。

- 静的Webアプリ
- Leaflet + OpenStreetMap系ベースマップ
- データはGeoJSON/JSON/CSV
- バックエンドなし
- ログインなし
- モバイルファースト
- 林道カードからGoogle Mapsの経路検索URLを開く
- 共有は公開URL（GitHub Pages等）

技術変更は、今週末の完成確率を明確に上げる場合のみ行う。

## 4. 自律行動ルール

- まずリポジトリ内の文書とデータを読む
- ユーザーに細かい確認を連発しない
- 破壊的変更、認証情報、費用発生、公開範囲の判断が必要な場合のみ確認する
- それ以外は合理的な既定値を選び、作業を前へ進める
- 長い作業は `PLANS.md` に実行計画を更新してから進める
- 各タスク終了時に `STATUS.md` と `TASKS.md` を更新する
- 途中成果物でも動作確認できる状態を保つ

## 5. プロジェクトがブレないためのルール

新しい提案は次の3分類で扱う。

- 🟢 採用推奨: 今週末MVPに必要。原則実装する
- 🟡 保留: 良い案だが今週末は実装しない。`docs/DECISIONS.md` に記録
- 🔵 将来構想: 長期構想。今の作業を止めない

**提案を理由に現在タスクを止めない。**

## 6. 林道データの原則

AIはデータを創作しない。既存情報を整理・構造化する。

禁止:

- 架空林道の登録
- 存在未確認の林道を確定扱い
- 情報源の省略
- IDの無断振り直し
- 同一林道の重複登録

情報源優先順位:

1. 行政資料（森林整備計画、林道台帳、自治体資料）
2. OpenStreetMap
3. 公開GIS、国土地理院等
4. ブログ・個人サイト（補完のみ）

## 7. IDルール

形式:

`都道府県3文字コード-6桁連番`

例:

- `NGN-000001`
- `GNM-000001`
- `IBR-000001`

長野県コードは `NGN`。

既存IDは安定識別子として扱う。並び替えのために振り直さない。

## 8. Lv0マスター

必須列はこれだけ。

```csv
ID,林道名,取得元
```

Lv0では未舗装判定、難易度、距離、景観、通行可否、カルテは調べない。

## 9. 調査のモード

### 高速モード

- 速度優先
- 取得漏れ許容
- 取得できた情報だけ登録
- 後から育成前提

### 完全モード

- 網羅性最優先
- 行政資料等を可能な限り横断
- 取得漏れを最小化

現在の長野県104件は**高速モード**。

## 10. データ更新時の必須検証

Lv0 CSVを変更したら実行:

```bash
python scripts/validate_lv0.py data/raw/NGN_Lv0_Master_高速モード_v0.1.csv
```

失敗した状態で完了扱いにしない。

## 11. MVPの非目標

今週末までに次は作らない。

- 全国完全版
- Supabase移行
- Flutterアプリ
- ユーザー認証
- SNS機能
- 大規模投稿システム
- 完全オフライン地図
- AI自動巡回更新

## 12. 完了の定義

MVP完成とは、スマホで公開URLを開き、茅野・諏訪周辺の候補林道を地図で確認し、林道情報を開き、入口までナビを開始できること。
