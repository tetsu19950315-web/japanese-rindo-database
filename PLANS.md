## 2026-07-10 長野県全域・候補育成計画

### Goal

既存17件をすべて位置・行政根拠・路面手がかり・通行情報・注意事項付きへ育成し、既存Lv0 104件と最新の長野県計画を母集団として、長野県全域の実走調査候補を選定する。

### 調査モード

- Lv0追加: 高速モード。最新の県行政資料で明示された新規路線名のみを既存ID末尾へ追加する。
- 候補選定: 全県スクリーニング。全路線の完全カルテ化ではなく、行政資料と位置情報の両方が確認できる路線を優先して肉付けする。
- 既存17件: 全件肉付け。位置未特定の場合は推測で埋めず、未特定と明記する。

### 選定基準

1. 行政資料で林道として存在確認できる。
2. OSMまたは行政図から入口・代表点・線形を確認できる。
3. OSMの `highway=track`、`surface=gravel/ground/unpaved`、`tracktype` 等に未舗装の公開手がかりがある。
4. `access=no/private`、恒常的通行止め等の明確な除外情報がない。
5. 県内地域の偏りを抑え、北信・東信・中信・南信から候補を残す。

### Outputs

- 既存Lv0 CSV: 既存IDを保持し、最新計画で確認した新規路線だけ末尾追加
- `data/processed/nagano_candidate_master.csv`: 全県候補の監査可能な詳細表
- `data/processed/nagano_map_data.json`: 地図表示用データ
- `data/processed/nagano_routes.geojson`: 根拠付き線形
- `data/processed/nagano_shortlist.json`: 地域別・優先度付き実走調査候補
- `data/processed/nagano_candidate_master.xlsx`: 更新・絞り込み用ワークブック
- 既存アプリ: 全県候補を表示し、地域・優先度で絞り込み可能にする

### 完了条件

- 既存17件の全行に肉付け状態と不足項目が明示される。
- 新規Lv0 IDは既存 `NGN-000001`〜`NGN-000104` を変更せず末尾追加される。
- 全県候補はすべて行政出典URLと位置出典を持つ。
- OSM線形は名前一致と周辺参考を区別し、根拠のない線を確定扱いしない。
- Lv0検証、候補データ検証、公開用バンドル検証がすべて成功する。

## 2026-07-15 林道入口・複数アクセスポイント計画

代表点を入口として扱う現状を改め、Google Mapsナビに使える公道側の入口・アプローチ地点を林道本体と分離して管理する。

### 登録ルール

- 正規の林道IDは変更しない。
- 両端が舗装路または主要な公道へ接続する完抜け林道は、同じ林道IDに `E1` / `E2` の入口点を持たせ、地図上では「林道名①」「林道名②」と表示する。
- 片側が別林道・作業道などへ接続しアクセス性に偏りがある場合、アクセスしやすい側だけを推奨入口とし、反対側は出口・接続点として保持する。
- 入口不明時は代表点を入口座標へ自動代入しない。
- 入口座標は行政資料、OSM、国土地理院資料、現地GPSを根拠とし、推定と確認済みを区別する。
- Street Viewは外部確認リンクとして提供し、画像の取得・保存・自動解析は行わない。

### 実装項目

- [x] `entrances` 配列と入口状態・根拠・アクセス種別を地図データへ追加
- [x] 同一林道の複数入口を衝突せず表示できる内部 `pointId` を追加
- [x] 入口別の選択、距離、Google Mapsナビ、座標コピー、共有URLを追加
- [x] Street View確認リンクを追加
- [x] 入口未特定候補は代表点として明示し、入口ナビを無効化
- [x] 既存24件を入口候補・代表点へ分類
- [x] 公開バンドル検証とスマホ幅ブラウザ確認

### 完了条件

- 同じ林道IDに2つの入口点を登録・選択・共有できる。
- 入口未特定の代表点を「入口」と表示しない。
- Google Mapsナビは入口または公道側アプローチ地点がある場合だけ有効になる。
- Street View確認はAPIキー不要のGoogle Maps URLで外部表示する。
- 既存の路線表示、地域・優先度絞り込み、現地記録を壊さない。

- 猿ヶ入・九竜平線・大上線は、OSM接続道路の種別に基づきアクセスしやすい側のみを推定入口として登録。
