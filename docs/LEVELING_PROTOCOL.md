# LEVELING_PROTOCOL — 林道候補の育成レベル判定 v1.0

## 目的

OSM起点で作成した長野県内1,063件の候補を、根拠の量と信頼性に応じて段階的に育成する。
レベルは人気度や走りやすさではなく、データとして確認できている深さを示す。

## レベル定義

### Lv0 — 発見候補

- OSMまたは単一資料から名称を発見した段階
- 独立資料との照合が完了していない
- 存在、位置、同名路線の同一性を確定扱いしない

### Lv1 — 基本情報確認

次の両方を満たす。

- OSMとは独立した公開資料で、同名路線の存在を確認できる
- 市町村、代表位置、行政計画掲載、路線形状など、名称以外の基本情報を1項目以上持つ

行政資料だけで位置未特定の場合も、行政計画掲載と市町村が確認できればLv1とする。

### Lv2 — 根拠付きカルテ

Lv1を満たし、さらに次を満たす。

- 位置付きOSMと行政資料が、名称・地域から一意に対応する。または、独立した公開資料2系統以上が同一路線を示す
- 名称・市町村・位置に加え、行政上の整備情報、路線形状、延長、路面、アクセス、通行情報などの属性を2項目以上持つ
- 出典URL、照合方法、確認日を記録している

Lv2でも通行可能や未舗装を保証しない。未確認項目は未確認のまま残す。

## 証拠の扱い

信頼性の優先順:

1. 国・県・市町村の行政資料
2. OpenStreetMap
3. 国土地理院・公開GIS
4. ブログ、動画、ニュース、施設案内などの公開Web資料
5. SNS投稿

OSMの `source=*` タグは候補抽出の参考には使うが、OSMから独立した証拠数には数えない。
検索結果の見出し・要約だけで確認した資料はLv1の補助証拠に限定し、Lv2の直接証拠には数えない。

## 同名路線と曖昧一致

- 正規化後の同名候補が複数ある場合、資料に市町村・位置・接続先などの識別情報がなければ自動昇格しない
- 短い一般名は完全一致だけで自動昇格せず、地域情報または「林道」の明示を必須とする
- 同一路線と判明しても既存IDを振り直さず、統合候補として履歴を残す
- 部分一致、読みの推測、地名だけの一致は証拠にしない

## 成果物

- `data/processed/nagano_level_master.csv`: 1,063件の判定結果
- `data/processed/nagano_level_evidence.csv`: 候補ごとの証拠明細
- `data/processed/nagano_level_report.json`: 件数、根拠種別、保留理由の集計
- 元のLv0 CSV: `Lv` と照合監査列を更新
