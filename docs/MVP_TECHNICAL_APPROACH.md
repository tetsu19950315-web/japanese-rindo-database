# MVP_TECHNICAL_APPROACH — 既定実装案

## 方針

最短で「スマホ・共有・視覚的・ナビ」を実現する。

## 推奨構成

- Static site
- Vite または最小HTML/JS
- Leaflet
- GeoJSON/JSON
- CSSはモバイルファースト
- GitHub Pages等で公開

## 画面

### メイン

- 全画面地図
- 上部にタイトル/簡易フィルター
- 下部またはモーダルに林道カード

### 林道カード

- 林道名
- ステータス
- 一言要約
- 重要注意
- 最終確認日
- 情報信頼度
- 「入口までナビ」

## ナビURL

入口座標がある場合、Google Mapsの経路検索URLを組み立てる。

実装時はURLエンコードし、座標がない場合はボタンを無効化する。

## データ読み込み

- `routes.geojson` → 地図
- `karte.json` → カード
- IDで結合

## 段階的実装

1. 3件のダミーではなく、実在確認済みデータで地図を動かす
2. 10件以上へ拡張
3. 3〜5件のカルテ追加
4. 公開
5. スマホ実機確認

## やらないこと

- 先にバックエンドを作る
- 先に認証を作る
- 先に全国データを作る
- 先に高度なルーティングを作る
