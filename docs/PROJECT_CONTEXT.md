# PROJECT_CONTEXT — プロジェクト背景

## 原点

Google Mapsでは細い林道や林道名の情報が弱い。国土地理院は細い道を確認できるが、舗装・未舗装判定、視認性、ナビ連携が弱い。ブログ、OpenStreetMap、動画等に断片的な情報が散らばっている。

目標は、それらを一括して地図上で確認し、入口までナビできる仕組みを作ること。

長期的な理想像は米国のonXのようなサービスだが、現在は仲間内利用を最優先する。

## 重要な方針転換

当初は全国版・大規模DB設計・Supabase等の話まで広がったが、ユーザーから明確に優先順位が示された。

現在の原則:

**プロジェクトを前に進める。今週末に使えるものを作る。**

## データ育成モデル

- Lv0: 採番 + 林道名 + 取得元
- Lv1: 基本情報
- Lv2: カルテ
- Lv3: 実走確認
- Lv4: コミュニティ更新

全林道を最初から深く調査しない。

まず母集団を作り、必要な林道だけ成長させる。

## ChatGPT/Codexと人間の役割

AI:

- 情報収集
- 重複整理
- 構造化
- ソース管理
- カルテ生成
- ツール実装

人間:

- 実走
- 写真
- GPX
- 現地確認
- 仲間の使用感

## 重要な思想

AIは「事実を創作する」のではなく「既存情報を整理する」。
